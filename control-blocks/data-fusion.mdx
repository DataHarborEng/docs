# Data Fusion

**Combine, don't duplicate.**

Join APIs with data lakes into unified, governed Virtual APIs. No new infrastructure required.

## Fuse

Reference another Virtual API and combine data into a unified meta response. Each source maintains its own governance rules while contributing to a composite view — no data duplication, no pipeline orchestration.

```yaml
objects:
  enriched_customers:
    fuse:
      - source: crm-api           # Virtual API name
        object: customers
      - source: analytics-lake    # Virtual API name
        object: customer_metrics
    joinOn: customer_id
```

## Use cases

| Scenario | Sources |
|----------|---------|
| Customer 360 view | CRM API + analytics data lake + support tickets |
| Enriched transactions | Payment API + fraud signals + merchant data |
| Unified inventory | ERP system + warehouse API + supplier feeds |
