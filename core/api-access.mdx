# API Access

**Your Virtual API, one URL away.**

Every Virtual API is accessible over REST. How consumers connect depends on the [visibility level](/core/agents#visibility-levels):

| Visibility | Endpoint | Auth | Registry |
|---|---|---|---|
| **Disabled** / **Private** | `/fetch/{leaseId}` | API key required | No |
| **Unlisted Open** | `/relay/{leaseId}/{path}` | None | No |
| **Listed Open** | `/relay/{leaseId}/{path}` | None | Yes — discoverable in catalog |

## Private access (fetch)

Virtual APIs with **Disabled** or **Private** visibility use the fetch endpoint with an API key:

```
https://service.dataharbor.co/fetch/{leaseId}
```

Include the `x-haven-api-key` header — you can find your API key in the Virtual API dashboard.

```bash
curl -H "x-haven-api-key: YOUR_API_KEY" \
  https://service.dataharbor.co/fetch/b097a4d3-44e6-4490-9f03-104f4d636e20
```

## Open access (relay)

Virtual APIs with **Unlisted Open** or **Listed Open** visibility use the relay endpoint — no authentication required:

```
https://service.dataharbor.co/relay/{leaseId}/{path}
```

The `{leaseId}` is the unique identifier for your Virtual API. The `{path}` maps to your upstream API's route structure — the same object matching and controls apply automatically.

```bash
# List properties
curl https://service.dataharbor.co/relay/166d4e26-3d40-4e44-806a-e56b93ebb121/properties

# Get a specific property's issues
curl https://service.dataharbor.co/relay/166d4e26-3d40-4e44-806a-e56b93ebb121/properties/1/issues
```

**Unlisted Open** is ideal for sharing with specific partners — they need the URL, but no credentials. **Listed Open** goes further: your Virtual API appears in the DataHarbor registry and catalog, making it fully discoverable.

## Registry

Virtual APIs with **Listed Open** visibility are indexed in the DataHarbor registry:

- **Registry catalog**: [https://service.dataharbor.co/registry](https://service.dataharbor.co/registry)
- **OpenAPI spec** (with query information): [https://service.dataharbor.co/openapi/v1.json](https://service.dataharbor.co/openapi/v1.json)

## Governance still applies

Open access doesn't mean ungoverned access. Every request through the relay endpoint still enforces:

- All controls (redact, tokenize, hash, transforms)
- Geo restrictions
- Expiration and shutdown
- Full audit trail
