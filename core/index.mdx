# Build Virtual APIs

Once you have a source connected, DataHarbor lets you turn it into multiple governed views without cloning the data.

This section is organized around the lifecycle of a Virtual API:

1. Understand the Virtual API model.
2. Learn the request pipeline from object matching through formatting.
3. Define how access paths, delivery, and agents should work.
4. Connect this model back to the source-side docs when you need enrollment or normalization details.
